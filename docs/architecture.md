# Architecture Overview

The MVP is local-first and split into clear layers:

- `app/backend/`: FastAPI endpoints and request/response orchestration.
- `app/services/`: core business logic (ingestion, chunking, retrieval, QA, comparison).
- `app/models/`: shared API schemas.
- `app/core/`: environment configuration and app-level settings.
- `app/frontend/`: Streamlit demo UI.
- `evals/`: lightweight evaluation scripts and sample dataset.

## Data flow

1. PDF upload goes to `/documents/upload`.
2. `PDFIngestionService` extracts per-page text.
3. `TextChunker` creates overlapping chunks.
4. Chunks are stored in SQLite (`documents`, `chunks` tables).
5. Chunk embeddings are generated by provider abstraction and indexed in FAISS.
6. Questions query FAISS, retrieve top-k chunks, and return an answer with citations.
